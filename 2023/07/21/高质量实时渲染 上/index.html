动机

今天，计算机图形学能够生成逼真的图像，例如复杂的几何图形、照明、材料、阴影，计算机生成的电影/特殊效果（很难或不可能区分真实和渲染……）。

但是精确的算法（特别是光线追踪）非常缓慢，所以它们被称为离线渲染方法。

通过适当的近似，我们可以产生合理的结果，但运行速度要快得多。

  • 对真实性进行了一些调整（阴影贴图，累积缓冲器）。

  3. 取平均数以获得可见度，例如：0.667

在RTR中，人们寻求简单而快速的解决方案，以解决一个反弹的间接照明问题

    • 然后把所有的表面斑块的贡献加起来

Reflective Shadow Maps (RSM)

  • 问题1：哪些surface patches是直接被照亮的
    • 用一个经典的shadow map完美地解决了这个问题
    • shadow map上的每个像素都是一个小的surface patch
  • 每个像素的准确出射radiance是已知的
    • 但只知道到摄像机的方向
  • 假设
    • 任何反射体都是diffuse的
    • 因此，出射的radiance对所有方向都是均匀的

  • 问题2：每个表面贴片的贡献是什么？
    • 对patch所覆盖的实体角度的积分
    • 可以转换为对patch面积的积分

（是四次方，因为 没有归一化，乘了两次）

不是RSM中的所有像素都能做出贡献

  • 可见性（仍然很难处理）
  • 方向
  • 距离

  • 加速
    • 理论上，shadow map中的所有像素都可以为p贡献
    • 我们可以减少这个数字吗？
    • 提示：PCSS的步骤1和步骤3
  • 采样来拯救

在一个RSM中需要记录什么？Depth, world coordinate, normal, flux, etc.

通常用于视频游戏中的手电筒

  • 优点
    • 易于实施
  • 缺点
    • 性能与lights的数量成线性关系
    • 对间接照明没有可见性检查
    • 许多假设：diffuse reflectors，深度为距离等。
    • 采样率/质量的权衡

Light Propagation Volumes（LPV）

关键问题：任何shading point查询来自任何方向的radiance

关键概念：radiance沿直线传播且不发生变化

关键解决方案：使用三维网格将radiance从直接照射的表面传播到其他任何地方

使用三维网格将辐射从直接照射的表面传播到其他任何地方

步骤

  1. 生成：生成radiance point set场景表示
    • 这是为了找到直接发光的表面，简单应用RSM就足够了！
    • 可以使用一组缩小的漫反射表面贴片（虚拟光源
  2. 注入：将虚拟光源点云注入到radiance volume中
    • 预先将场景细分为3D网格，为每个网格单元找到封闭的虚拟光源
    • 汇总它们的方向辐射分布，投影到前2阶SH（共4阶）
  3. 传播：Volumetric radiance传播
    • 对于每个网格单元，收集从其6个面接收到的radiance。求和，并再次使用SH表示
    • 重复多次，直到体积稳定为止
  4. 渲染：场景照明与最终光传播体积
    • 对于任何阴影点，找到它所在的网格单元，获取网格单元中的入射radiance（来自各个方向）
    • 着色

有问题吗？提示：查看p点

Voxel Global Illumination (VXGI)

  • 仍然是两步算法

  • 与RSM的两个主要区别

    • 直接照射像素->（分层）体素
    • 在RSM上采样 -> 在3D中追踪反射锥体（注意RSM采样的不准确性）

  • 将整个场景虚拟化，建立层次结构

步骤：

  1. 从光出发
    • 在每个体素中存储入射分布和正态分布，更新层次结构
  2. 从相机出发
    • 对于glossy表面，朝反射方向跟踪1个圆锥体，根据锥体的（增长）大小查询层次结构
  3. 对于diffuse，跟踪几个锥体（例如8个）

屏幕空间定向遮蔽（SSDO）

Screen Space Directional Occlusion

什么是SSDO？SSAO的改进，考虑（更多）实际间接照明

关键概念：为什么我们必须假设均匀的入射间接照明？我们已经知道间接照明的一些信息！

听起来熟悉吗？（RSM）

SSDO利用渲染的直接光照（不是来自RSM，而是来自相机）

与路径跟踪非常相似

  • 在着色点 ，发射随机光线
  • 如果没有碰到障碍物，直接照射
  • 如果碰到障碍物，则为间接照明

与SSAO的比较

  • AO：间接照明 + 无间接照明（在一定半径内，计算红色圈忽略橙色圈）
  • DO: 无间接照明 + 间接照明（与路径跟踪相同）（在一定半径内，计算橙色圈忽略红色圈）

分别考虑未闭合和闭合方向

像素（patch）的间接光照度由上一课得出

与HBAO类似，测试样本在局部半球的深度

SSDO：质量更接近离线渲染

问题：仍然是短距离内的GI、可见性、屏幕空间问题（看不到的表面信息缺失）

屏幕空间反射（SSR）

Screen Space Reflection

什么是SSR？在RTR中引入全局光照的一种方法，执行光线追踪，但不需要三维基元（三角形等）

SSR的两个基本任务

  • 交点：任何光线与场景之间的交点
  • 着色：相交像素对着色点的贡献

基本 SSR 算法 - 镜像反射

对于每个片段

  • 计算反射光线
  • 沿射线方向跟踪（使用深度缓冲器）
  • 使用交叉点的颜色作为反射颜色

直线运动

目标：找到交点

  • 每一步检查深度值
  • 质量取决于步长
  • 可以细化

产生深度Mip-map（使用最小值来代替平均值）

为什么选择深度Mipmap

  • 与 3D 中的层次结构 (BVH, KD-tree) 非常相似
  • 能够更快地剔除一堆中的不相交部分
  • min操作保证了一个保守的逻辑
    • 如果一条射线没有与一个较大的节点相交，那么它将永远不会与它的任何子节点相交。

分层追踪：min-Z金字塔的stackless ray walk

shading使用SSR

  • 与路径追踪完全没有区别
  • 再次假设diffuse反射/辅助光源

问题

  • 是否会引入平方距离衰减？
  • 它能处理遮光点和辅助光源之间的遮挡吗？

我们的要求

  • 清晰和模糊的反射
  • contact hardening（shading point往外打为锥形，越传播越大，收集对shading point影响的范围也越来越大）
  • 镜面拉伸（表面各向同性）
  • 每像素roughness和法线

我们的提高

  • 知道BRDF和观察的方向，没有必要均匀的从四面八方采样

  • 从附近的点tracing的信息复用

把屏幕空间的图预先模糊一遍，便于查询（但屏幕空间的图深度不同）

总结

优点

  • 对光泽和镜面反射具有快速性能
  • 质量好
  • 无尖峰和闭塞问题

缺点

  • 在diffuse反射情况下效率较低*。
  • 屏幕外信息缺失
